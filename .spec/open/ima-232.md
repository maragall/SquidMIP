# IMA-232 — Package app

_slug: package-app_

## Why
Demoers download one binary and read TB in place ($0 hosting).

## Scope
arm64 `.app` via PyInstaller, **notarized**, published to GitHub Releases on tag.
This ticket is repair-and-finish, not greenfield: the build spec + CI already
exist (from IMA-185, commit `03c5246`) and are currently RED on `main` — the
freeze job has never run because its `needs: smoke` gate fails.

## Acceptance / oracle
The `.app` launches and opens a dataset on a clean machine — **verified in CI**
by two legs: a GUI launch (offscreen, asserts the process stays alive past Qt
plugin load) and a headless CLI run that writes a plate from a fixture.

## Depends on
none. **One human action before merge:** create an Apple Developer account
($99/yr) and add the signing secrets (see D2). The workflow is written to consume
them; without them the notarize step warns and skips, and the app fails its
oracle on a clean Mac.

**Unstated assumption, now stated:** every demo machine is Apple Silicon. An
Intel-Mac demoer gets a binary that cannot run under any translation path. If
that assumption is wrong, universal2 moves from "not in scope" to P1.

### D2 — Signing: codesign + notarize + staple (macOS)
- Keychain setup step first: `security create-keychain`, import the p12,
  `security set-key-partition-list` (without this, `codesign` hangs on a UI prompt
  in CI).
- Sign **inside-out**: nested `.dylib`/`.so`/frameworks first, then the `.app`.
  Do **not** rely on `codesign --deep` — Apple deprecates it and it misses
  PyInstaller's nested binaries.
- **Entitlements plist is required** and must include
  `com.apple.security.cs.allow-unsigned-executable-memory` and
  `com.apple.security.cs.disable-library-validation` — both are needed for a
  hardened-runtime PyInstaller + PyQt5 bundle to start at all.
- Secrets (five, not three): `MACOS_CERT_P12` (base64), `MACOS_CERT_PASSWORD`,
  `MACOS_TEAM_ID`, `MACOS_APPLE_ID`, `MACOS_NOTARY_PASSWORD` (app-specific) — or
  an App Store Connect API key trio in place of the last three.
- Signing/notarizing is skipped with a loud warning when secrets are absent, so
  forks still build an unsigned dev artifact.
- `pty.fork` in `_Terminal._start` (`_viewer.py:210-221`) is fine under hardened
  runtime as written — the child does only `chdir` + `execvpe`.

### D3 — Distribution: GitHub Release on tag
- `release` job gated on `startsWith(github.ref, 'refs/tags/v')` using
  `softprops/action-gh-release`.
- **Archive with `ditto -c -k --keepParent` before any upload.** GitHub's
  `upload-artifact` re-zips and discards symlinks and exec bits, which destroys
  Qt `.framework` internals — the downloaded `.app` would not launch even if
  perfectly notarized. This applies to the per-push artifact too.
- Re-create the zip **after** stapling (the ticket staples onto the `.app`).

### D4 — Build spec: commit `hcs-viewer.spec`
- Generate a PyInstaller `.spec`, drop the conflicting `-c` (confirmed: both `-c`
  and `--windowed` write PyInstaller's `console` dest, last flag wins, and
  `console=True` suppresses the macOS `.app` bundle).
- Host `bundle_identifier` (`com.cephla.hcs-viewer`), icon, and the D2 entitlements
  reference. CI runs `pyinstaller hcs-viewer.spec`.

### D5 — Frozen binary is its own CLI *(revised — outside voice)*
The argv dispatch alone does **not** fix this. Three parts:

1. **Disambiguate argv — do not break existing behavior.** `_viewer.main()`
   (`:2069-2070`) already reads `sys.argv[1]` as a dataset path to open in the GUI,
   and macOS Finder "Open With" / drag-onto-dock passes a path exactly that way.
   Rule: `argv[1]` is an existing directory → **GUI**, open that dataset (preserved);
   otherwise → **CLI** dispatch to `squidmip._cli:main`. Bad args → nonzero exit.
2. **Rewrite the banner.** `_viewer.py:1401-1409` prints `python -m squidmip …`.
   A frozen bundle has no `python` and no `squidmip` on disk — the executable is
   `hcs-viewer`. When frozen, the banner must show the resolved app-binary form.
   Aliasing `python` is *not* a fix: `hcs-viewer -m squidmip …` feeds `-m squidmip`
   into the pydantic CLI and dies.
3. **Populate `setup_cmds`.** It is hardcoded `[]` at `:1414`. When frozen, emit
   `alias squidmip='<sys.executable>'` (and the `doskey` equivalent) so the banner's
   commands actually resolve in the pane.

### D6 — Simplify the smoke import list *(revised — outside voice)*
- Replace the literal MODULES list with a **`pkgutil` walk of `squidmip` only**.
  Drop the `importlib.metadata` + dist-name→import-name exception table from the
  original plan: that table is the same hand-maintained list relocated, and the
  `pytest` step in the same job already imports every third-party dependency
  transitively. This keeps the user's "derive, don't hand-write" intent without
  the machinery.
- **Also decouple**: `freeze` should no longer be silently skipped by a smoke
  failure. Keep smoke as a required check, but let `freeze` run so a stale gate
  can never again withhold every artifact.

### D7 — Raise Python floor to 3.11
- `pyproject.toml`: `requires-python = ">=3.11"`. Drop the `3.10` row from
  **`ci.yml`** (not `build.yml`, which already pins 3.11). Fix `README.md:16`.

### D8 — Extract `_TerminalBase` *(kept, but de-sequenced — outside voice tension)*
- Outside voice argued this is refactor-for-its-own-sake: the banner is built once
  in `_build_cli_tab` and passed to both classes, so the only thing D5 duplicates
  is the one-line PATH prepend at `:208` and `:348`, and the two classes differ
  materially (pty/fcntl vs QProcess).
- **Resolution:** keep the extraction (it was an explicit decision), but **do the
  D5 fix first and refactor after**, so the real fix is never blocked behind a
  structural change. Downgraded P2 → P3.

### D9 — CI launch test *(revised — two legs, outside voice)*
The original single leg tested only the CLI path and could not call the fixture.

- **Extract a callable builder.** `squid_dataset` is a `@pytest.fixture` taking
  `tmp_path` (`tests/conftest.py:81`); modern pytest raises if you call it directly.
  Pull the body into `build_squid_dataset(root) -> (root, arrays)` and have the
  fixture delegate to it. The CI test imports that function.
- **Leg 1 — GUI launch.** Start the frozen `.app` with `QT_QPA_PLATFORM=offscreen`
  and a timeout; assert it stays alive past Qt plugin load, then terminate cleanly.
  This is the leg that catches Qt cocoa-plugin and dylib resolution failures — the
  dominant frozen-GUI failure mode, which the CLI path does not exercise at all.
- **Leg 2 — CLI on a fixture.** Run the frozen binary headless against the built
  dataset; assert exit 0 and `plate.ome.zarr` written (`squidmip/_output.py:301`).
- **Use compressed TIFFs in the CI fixture.** `conftest.py:77` calls
  `tifffile.imwrite(...)` with no compression, so `imagecodecs` is never exercised —
  a missing `imagecodecs` in the bundle would pass this test and then fail on real
  Squid data (which is LZW-compressed). Write at least one LZW plane.
- **Verify Gatekeeper explicitly.** The CI-built binary carries no quarantine
  attribute, so "works on a clean machine" stays unproven unless the job also runs
  `xcrun stapler validate` and `spctl -a -vv` against the stapled `.app`.

### P3 — Trim `--collect-all` *(revised — outside voice)*
- Drop `imageio_ffmpeg` (confirmed: no `imageio` import anywhere in `squidmip/`;
  pulls a ~40MB ffmpeg binary).
- **Keep** `--collect-all ndviewer_light` unless someone verifies it ships no
  package data files — `ndviewer_light.core` is imported lazily (`_viewer.py:1448`),
  and collect-all is what guarantees data files come along.
- Keep `--collect-all` for `tensorstore` and `ndv` (dynamic binaries/plugins).
- Confirm `imagecodecs` is bundled via the `pyinstaller-hooks-contrib` hook; D9's
  compressed-TIFF leg is the assertion that proves it.

## Test plan

New tests (headless, `QT_QPA_PLATFORM=offscreen`):

- `tests/test_entry.py` — `hcs_viewer_entry` dispatch:
  - argv empty → GUI path
  - argv[1] is an existing directory → GUI opens that dataset (**regression guard**
    for Finder "Open With" / drag-onto-dock)
  - argv[1] is not a directory → `squidmip._cli:main` called
  - bad args → nonzero exit
- `tests/test_viewer.py` (extend) — frozen-aware CLI resolution:
  - frozen (`sys.frozen` monkeypatched) → app-binary form, and `setup_cmds` emits
    the alias
  - source → `python -m squidmip` form
  - banner text matches the resolved form (**regression guard** for the frozen bug)
- `tests/test_smoke_import.py` — the `pkgutil` walk enumerates every `squidmip.*`
  submodule; a deliberately-missing module still exits nonzero
- `tests/conftest.py` — `build_squid_dataset()` extracted; existing fixture
  delegates (no behavior change to the ~2500 lines of existing tests)
- **CI-only** (`build.yml`) — the two-leg launch test + `spctl`/`stapler validate`

### Failure modes
| Codepath | Realistic prod failure | Test? | Error handling? | Silent? |
|----------|------------------------|-------|-----------------|---------|
| notarize | secrets absent / cert expired | D9 `spctl` check | warn + skip | no |
| argv dispatch | Finder passes a path → runs headless instead of opening GUI | test_entry (regression) | directory check | **was silent** |
| frozen Qt load | cocoa plugin not collected → crash on launch | D9 leg 1 | crash | **was silent; now caught** |
| frozen decode | `imagecodecs` missing → real LZW data fails | D9 leg 2 (compressed) | exception | **was silent; now caught** |
| artifact upload | symlinks/exec bits stripped → `.app` won't launch | D9 runs on the archived artifact | — | **was silent** |

Critical gaps closed: four (frozen Qt load, frozen decode, argv regression,
archive corruption) — all previously silent.

## NOT in scope

- Universal2 (x86_64+arm64) — arm64 only per spec. **Revisit if any demoer is on
  an Intel Mac**; the binary cannot run there under any translation path.
- Homebrew cask / installer packaging — GitHub Release is the channel.
